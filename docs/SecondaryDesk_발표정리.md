# SecondaryDesk — 사용 시나리오 & 사용 스펙 (발표 정리)

> VC 구주(Secondary) 세일즈 CRM — 만기 펀드 EXIT를 위한 구주 세일즈 활동 관리 내부 도구
> 이 문서는 사내 발표용 PPT 가공을 위한 원본 정리본입니다. 섹션 단위로 슬라이드화하세요.

---

## 0. 한 장 요약 (Executive Summary)

- **무엇**: 보유 포트폴리오사의 구주를 후속 투자사에 파는 영업(B2B Sales)을 구조화한 사내 CRM.
- **왜**: 펀드 만기 → EXIT가 핵심 과업인데, 컨택·진척이 개인 네트워크에 흩어져 휘발됨.
- **어떻게**: "매물(구주) × 투자사"를 **딜**로 정의해 파이프라인·타임라인·대시보드로 축적·가시화.
- **차별점**: ① 외부 데이터 자동 연동(sparkERP/slab, DIVA 공시) ② 드라이파우더·EXIT 손익·재무건전성을 **추정/시뮬레이션** ③ Claude AI로 재무제표 자동 추출.
- **상태**: 실서비스 운영 중(2026-06 기준). Next.js + Supabase, Vercel(서울 리전) 자동 배포.

## 1. 문제 & 해결 (Before → After)

| 구분 | As-Is (도입 전) | To-Be (SecondaryDesk) |
|---|---|---|

| 진척 공유 | 누가 어디까지 하는지 모름 | 딜 보드(칸반)에서 **팀 전체 한눈에** |

| 외부 정보 | 수기 취합 | sparkERP·DIVA·재무제표 **자동 수집/추출** |
| 의사결정 | 감 | 대시보드·전환율·EXIT 시뮬레이션 |

## 2. 핵심 컨셉 — B2B CRM 객체를 구주 세일즈에 매핑

| 일반 CRM | SecondaryDesk | 의미 |
|---|---|---|
| Account | **투자사 (Investor)** | 구주 매수 후보 (VC/CVC/PEF/세컨더리 등) |
| — | **조합 (Fund)** | 투자사가 운용하는 펀드 (드라이파우더·주목적·만기) |
| Contact | **컨택 (Contact)** | 투자사 내 의사결정자/심사역 |
| Product | **매물 (Listing)** | 우리가 파는 포트폴리오사 구주 |
| **Opportunity** | **딜 (Deal)** | **매물 × 투자사 교차점 = 세일즈 1건** ⭐ |
| Activity | **활동 (Activity)** | 컨택 이력 타임라인 |

> 핵심 통찰: **딜**은 "어떤 매물을 어떤 투자사에 파는가"라는 다대다(N:M)를 해소하는 중심 객체.
> 모든 파이프라인·타임라인·대시보드가 딜을 축으로 돈다.

## 3. 사용자 & 권한

| 페르소나 | 니즈 | 주 사용 기능 |
|---|---|---|
| **심사역 (member)** | 본인 딜 진행·이력 기록·네트워크 활용 | 딜 보드, 활동 기록, 투자사/조합/매물 |
| **팀 리드 (lead)** | 전체 진척·자원 배분 | 대시보드, 구성원 관리, 딜 재배정 |

- **멀티유저 공용 자산**: 투자사·조합·컨택·매물은 팀 공유. 딜에는 담당자(owner) 존재.
- **승인제 가입**: 회원가입 → 리드 승인 후 접근(대기 상태 분리). 비로그인 전면 차단.
- **보안**: Supabase RLS로 인증 사용자만 접근. 구주·재원 정보는 민감정보(MNPI 성격)로 외부 비공개.

## 4. 사용 시나리오 (Usage Scenarios)

### 시나리오 A — 새 투자사 발굴부터 딜 생성까지 (심사역)
1. **딜 보드 → `+ 딜 생성`**: 새 투자사(또는 기존 선택) + 소개할 매물 1개 이상 선택 → 매물마다 딜 자동 생성.
2. 생성된 딜이 파이프라인 **`컨택` 단계**에 등장. 단계 진입 일자(컨택일자) 지정 가능(과거 딜 소급 입력).
3. 미팅·통화 후 **투자사 상세 → 활동 기록**(유형·일시·컨택·내용) 1클릭 저장 → 컨택 `최근 컨택일` 자동 갱신.
4. 진척에 따라 카드를 **드래그로 단계 이동**(컨택 → 기업소개 → IR·실사 → 클로징, 무산 시 드랍).
5. `다음 액션 예정일`이 지나면 카드에 시각 경고 → 후속 관리.

### 시나리오 B — 매물을 여러 투자사에 동시 소개 (세일즈 캠페인)
1. **딜 보드 → `+ 딜 생성`**: 투자사 고정 + **매물 복수 선택**(또는 저장한 "즐겨찾기 묶음" 적용) → 한 번에 다건 딜 생성.
2. 운용펀드로 매물을 먼저 구분·검색해 묶음 구성.
3. 보드 상단 **매물·담당자 검색/필터**로 캠페인 진척 추적.

### 시나리오 C — 후속 투자사 매칭 (조합 데이터 활용)
1. **조합 탐색** 탭에서 전체 투자사(GP)의 운용 조합을 **결성일 최신순**으로 열람 → 최근 결성(드라이파우더 보유 가능성 높은) 조합 식별.
2. 조합의 **드라이파우더(추정)**·투자분야·목적구분으로 매수 여력 가늠 → 해당 투자사 상세로 이동해 딜 생성.

### 시나리오 D — EXIT 가격 시뮬레이션 (투자 검토/내부 보고)
1. **EXIT Simulator**에서 운용펀드 → 매물 선택 → 라운드 데이터 자동 로드(slab 연동).
2. **할인율별 매각 총액·손익·MoM**과 손익분기 할인율 확인.
3. **다음 라운드 예상 단가** 입력(또는 **밸류에이션으로 추정**: 포스트머니·모집금액·발행주식수) → 현재 vs 다음 라운드 **비교 차트** 표시. 발행주식수는 slab에서 자동 반영.

### 시나리오 E — 매물(포트폴리오사) 재무 건전성 점검
1. **재무 점검** 탭에서 매물 선택 → slab 재무 또는 업로드한 재무제표(PDF/이미지)를 **Claude가 자동 추출**.
2. 매출·순이익·현금·자본 등 11개 값을 분기 시계열로 저장 → **런웨이·자본잠식률 등 건전성 판정**.

### 시나리오 F — 외부 데이터 일괄 적재 (운영/관리)
1. **가져오기**: DIVA(벤처캐피탈협회 공시) 조합 CSV 업로드 → 운용사ID 기준 투자사·조합 멱등 적재(공동GP 분해).
2. **명함/컨택 임포트**, **딜 엑셀 임포트**도 동일 마법사(컬럼 자동 매핑)로 처리.
3. **운용펀드 → ERP 동기화**: sparkERP 펀드·회사·투자내역·재무를 매일 자동 크론 + 수동 동기화.

## 5. 기능/탭별 사용 스펙 (Feature Spec by Tab)

> 좌측 사이드바 메뉴 순서대로. (맨 위 **알아보기**(`/guide`)는 각 메뉴의 사용법을 팀원에게 안내하는 온보딩/도움말 페이지입니다.)

### 5.1 대시보드 (`/`)
- 파이프라인 현황(단계별 딜 수·합산 예상금액), **단계 전환율**, **IR·실사 도달 Top3 매물**, 정체(stalled) 딜, 투자사별 진척.
- **운용펀드 필터**: 펀드 선택 시 해당 펀드 보유 매물·관련 딜만 집계.

### 5.2 딜 보드 (`/deals`) ⭐ 메인 작업 공간
- **칸반 파이프라인**: 컨택 → 기업소개 → IR·실사 → 클로징 (+ 드랍 — 종료딜 상시 표시).
- 드래그 단계 이동 즉시 저장(단계 이력 기록). 단계별 음영 패널로 경계 구분.
- 카드: 매물명·투자사명·담당자·예상금액·다음 액션(+예정일, 임박 시 경고).
- 상단 **매물·투자사 검색/필터**, **운용펀드 필터**, `+ 딜 생성`(매물 복수 + 즐겨찾기 묶음).
- **딜 일괄 가져오기** (`/deals/import`): 엑셀 임포트.

### 5.3 투자사 (`/investors`, `/investors/[id]`)
- 목록: 투자사별 **조합 수·결성총액 합·매칭 딜 수** 표기/정렬.
- 상세(Related Lists 통합 뷰): 개요 · **운용 조합**(결성일·만기·목적·투자분야·결성총액·**드라이파우더 추정**) · 컨택 · 연결된 딜 · **활동 타임라인**.
- 드라이파우더: 결성일·존속기간 기반 S커브 추정(실측 입력 시 실측 우선).

### 5.4 조합 탐색 (`/associations`) — 신규
- 전체 투자사(GP)의 운용 조합을 한 페이지에 **집계**, **결성일 최신순** 정렬.
- **조합명·투자사 실시간 검색 필터**, 최근 3개년 결성 하이라이트, 드라이파우더 추정 동일 표기.
- 데이터 출처: DIVA 공시 임포트분.

### 5.5 매물 (`/listings`, `/listings/[id]`)
- 포트폴리오사 구주를 매물로 등록(회사명·상태 중심, 가벼운 정보).
- **운용펀드 태깅**(N:M, 중복 소속 허용). 상태/운용펀드 필터.
- 상세: 소속 운용펀드 · 연결된 딜(검토 중 투자사) · **slab ERP 원본 주식정보**(발행주식수·주당가격·밸류에이션·투자내역).
- **매물 엑셀 임포트** (`/listings/import`).

### 5.6 운용펀드 (`/funds`, `/funds/erp`, `/funds/sync`)
- 우리가 운용·매각하는 펀드 마스터(약칭·만기·상태). ※ 투자사의 매수측 조합과 별개 개념.
- **ERP 동기화**(`/funds/sync`): sparkERP 펀드/회사/투자내역 제자리 동기화(영업값 보존, 미매칭 수기 매칭).
- **ERP 조합 현황**(`/funds/erp`).

### 5.7 EXIT Simulator (`/exit-scenario`)
- 운용펀드 → 매물 선택 → 라운드 데이터 로드. **할인율(0~40%)별 매각 단가·총액·손익·MoM**, 손익분기 할인율 강조.
- 펀드별 결과 매칭(공동 보유 시), 요약 카드(원금·예상가치·보유주식수·손익분기).
- **다음 라운드 단가 비교**: 직접 입력 또는 **밸류에이션 추정**((포스트머니 − 모집금액) ÷ 발행주식수). 발행주식수 **slab 자동 조회**. 현재 vs 다음 라운드 **그룹 막대 비교 차트**.

### 5.8 재무 점검 (`/financials`)
- slab 재무 또는 업로드 재무제표(PDF/이미지)를 **Claude로 자동 추출**(매출·순이익·현금·자본 등 11개 값, 분기 시계열).
- 보유현금·런웨이·자본잠식률 등 파생 지표로 **건전성 판정**.

### 5.9 가져오기 (`/import`, `/import/funds`, `/import/contacts`)
- 공용 임포트 마법사(컬럼 자동 매핑 + 미리보기 + 템플릿 다운로드).
- DIVA 조합 / 명함·컨택 / 딜 / 매물 일괄 적재. 멱등 키 기반 중복 방지.

### 5.10 구성원 (`/members`, lead 전용) · 검색 (`/search`) · 알림 (`/notifications`)
- 구성원 관리·승인, 전역 검색, 액션 리마인더.

## 6. 데이터 모델 (요약)

```
User        1─N Deal (owner)
Investor    1─N Fund        (매수측 조합, DIVA/수기)
Investor    1─N Contact
Investor    1─N Deal ─N Activity
Listing     1─N Deal
Listing     N─M HoldingFund (ListingFund 태그 / 우리 운용펀드)
Listing     1─N ExitScenarioRound  (EXIT 라운드 단가·주식수)
Listing     1─N FinancialStatement (분기 재무 시계열)
```

- 핵심 객체: **투자사 · 조합 · 컨택 · 매물 · 운용펀드 · 딜 · 활동**.
- 외부 연동 키: `funds.diva_asct_id`(DIVA 조합ID), `investors.diva_oper_inst_id`(DIVA 운용사ID), `listings.bubble_id`·`holding_funds.bubble_id`(sparkERP _id).

## 7. 외부 연동 & AI

| 연동 | 대상 | 용도 | 방식 |
|---|---|---|---|
| **sparkERP (slab / Bubble)** | 회사·펀드·투자내역·재무·주식정보 | 매물·운용펀드 자동 보정, EXIT 라운드 자동채움, 발행주식수, 재무 원본 | 매일 자동 크론(`/api/cron/erp-sync`) + 수동 동기화 |
| **DIVA (벤처캐피탈협회 전자공시)** | 전체 벤처투자조합·운용사 | 투자사·조합 일괄 적재 | Playwright 로컬 스크래퍼 → CSV 임포트(멱등) |
| **Claude API (Anthropic)** | 재무제표 PDF/이미지 | 11개 재무값 자동 추출 → 건전성 판정 | 서버 액션에서 호출 |

## 8. 기술 스택 & 아키텍처

| 레이어 | 사용 기술 |
|---|---|
| 프레임워크 | **Next.js 16 (App Router, Turbopack) + React 19 + TypeScript** |
| UI | **Tailwind CSS v4 + shadcn/ui (radix-ui)**, lucide 아이콘, 자체 테마(라이트/다크) |
| 상호작용 | dnd-kit(칸반 드래그), xlsx(엑셀 임포트) |
| DB/인증/스토리지 | **Supabase** (Postgres + Auth + **RLS** + Storage) |
| AI | **@anthropic-ai/sdk** (Claude) |
| 배포/운영 | **Vercel** (서울 리전 `icn1`, GitHub main 푸시 자동 배포, 매일 크론) |

- 서버 컴포넌트 + 서버 액션 중심(풀스택 단일 코드베이스), 인증 가드는 Proxy(구 Middleware).
- 성능 최적화: 서울 리전 이전(태평양 왕복 제거), 딜 보드 쿼리 다이어트, 명함 검색 디바운스 + pg_trgm 인덱스.
