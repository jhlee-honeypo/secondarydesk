# sparkERP(Bubble) 데이터 연동 요청서

> **받는 분:** sparkERP(Bubble) 앱 관리자님
> **보내는 목적:** 저희 사내 시스템(SecondaryDesk, 구주 세일즈 CRM)에서
> sparkERP의 데이터를 자동으로 가져와 가공할 수 있도록, **데이터를 외부에서 읽을 수 있게**
> 열어주시길 부탁드립니다.
> **작성:** 2026-06-16

---

## 한눈에 보는 요청 (가장 중요)

> sparkERP에 지금 **company(포트폴리오)·fund(조합)·quarterlyupdate(분기현황)** 3개 데이터는
> 이미 외부에서 잘 읽히고 있습니다. 감사합니다 🙏
>
> **나머지 데이터 타입들(특히 투자내역 등)도 "이 3개와 똑같은 방식으로" 열어주시고,
> 마지막에 [Deploy](배포) 버튼을 눌러 실제 사이트에 반영**해 주시면 됩니다.
>
> 어려운 설정이 아니라, **이미 해두신 설정을 다른 데이터에도 똑같이 복제**하는 작업입니다.

---

## 1. 왜 이 요청을 드리나요 (배경)

저희는 sparkERP에 쌓인 정보를 저희 영업 시스템에서 활용하려고 합니다.

- **지금 당장 필요한 것:** 투자내역(투자금액·보유주식수) → 매각 시뮬레이션 계산에 사용
- **앞으로 하고 싶은 것:** 투자 라운드, 거래내역, 회사 지표 등 **sparkERP의 다양한 데이터를
  폭넓게 가져와 가공** (그래서 한 번에 여러 데이터를 열어주시길 부탁드립니다)

저희는 데이터를 **읽기(조회)만** 합니다. sparkERP의 데이터를 수정하거나 지우지 않습니다.

---

## 2. 지금 상태 (그림으로 이해하기)

Bubble 앱에는 **'미리보기(개발) 버전'** 과 **'실제(운영) 버전'** 두 가지가 있습니다.

| | 미리보기(개발) 버전 | **실제(운영) 버전** ← 저희가 쓰는 곳 |
|---|:---:|:---:|
| 외부에 열린 데이터 종류 | 13개 (대부분 열림) | **3개만** (company·fund·quarterlyupdate) |
| 투자내역(sparklabinvestment) | ✅ 열려있음 | ❌ **닫혀있음** |

즉, **미리보기 버전에는 이미 열어두셨는데, 실제 버전에는 아직 반영(배포)이 안 된 상태**입니다.
저희 시스템은 실제 버전을 사용하므로, **실제 버전에도 열려야** 데이터를 가져올 수 있습니다.

---

## 3. 열어주시면 좋은 데이터 목록

아래는 미리보기 버전에 있는 데이터 종류입니다. **가능하면 전부**,
어렵다면 ⭐ 표시된 것이라도 실제 버전에 열어주세요.

| 데이터 이름(영문) | 추정 의미 | 상태 |
|---|---|---|
| company | 포트폴리오 회사 | ✅ 이미 열림 |
| fund | 조합/펀드 | ✅ 이미 열림 |
| quarterlyupdate | 분기 현황 | ✅ 이미 열림 |
| ⭐ **sparklabinvestment** | **우리 투자내역(투자금액·보유주식수)** | ❌ 닫힘 |
| ⭐ fundinground | 투자 라운드 정보 | ❌ 닫힘 |
| ⭐ fundingroundinvestor | 라운드별 참여 투자자 | ❌ 닫힘 |
| ⭐ transaction | 거래(매매) 내역 | ❌ 닫힘 |
| share | 주식/지분 정보 | ❌ 닫힘 |
| companyhighlight | 회사 주요 지표 | ❌ 닫힘 |
| exchangerate | 환율 | ❌ 닫힘 |
| country | 국가 | ❌ 닫힘 |
| 그 외(batch, sp 등) | 내부용 추정 | ❌ 닫힘 |

> ⭐ 4개만이라도 우선 열어주시면 지금 필요한 작업은 시작할 수 있습니다.
> 보안상 일부만 열고 싶으시면 ⭐ 항목 위주로 부탁드립니다.

## 4. 따라 하기 (Bubble 화면 기준 단계)

> 핵심: **company/fund에 이미 해두신 것과 똑같이** 다른 데이터에도 해주시면 됩니다.

### 1단계 — 데이터를 API로 열기
- 상단 **Settings(설정) → API** 메뉴로 들어갑니다.
- 데이터 종류 목록이 나오고, 종류마다 체크박스가 있습니다.
- **company, fund 옆에 이미 체크돼 있는 것과 똑같이**, 위 ⭐ 항목들(가능하면 전부)에
  체크를 해주세요. (읽기 용도이니 "modify(수정)" 권한은 켜지 않으셔도 됩니다)

### 2단계 — 외부에서 볼 수 있게 권한 주기 (Privacy/프라이버시)
- 상단 **Data(데이터) → Privacy(프라이버시)** 메뉴로 들어갑니다.
- 종류마다 "누가 어떤 항목을 볼 수 있는지" 규칙이 있습니다.
- **company/fund에 설정해 두신 규칙과 동일하게**, 위 ⭐ 항목들도
  외부에서 항목을 볼 수 있도록(예: "Everyone else"가 "View all fields") 맞춰주세요.

### 3단계 — 실제 사이트에 반영 (가장 중요!)
- 화면 오른쪽 위 **Deploy(배포) 버튼** → **"Deploy current version to live"** 를 누릅니다.
- 이걸 눌러야 1·2단계 설정이 **실제(운영) 버전에 적용**됩니다.
  (안 누르면 미리보기 버전에만 적용돼서 저희가 여전히 못 읽습니다)

## 5. 토큰(비밀번호 같은 열쇠)에 대하여

전에 보내주신 토큰(`3611ea3f...0e2521`)은 저희가 테스트해보니 **Bubble이 받아주지 않았습니다**
(만료됐거나 다른 용도의 값으로 보입니다).

**하지만 토큰은 없어도 괜찮습니다.** 지금 company·fund가 토큰 없이도 잘 읽히는 것처럼,
위 4번대로 열어주시면 **토큰 없이 바로 동작**합니다.

만약 보안상 "아무나 말고 우리 시스템만 읽게 막고 싶다" 하시면, 그때는
**Settings → API에서 새 토큰(API Token)을 발급해 저희에게 전달**해 주시면 됩니다.
(이 경우 위 2단계 권한을 "토큰을 가진 요청만 허용"으로 잡으시면 됩니다)

## 6. 다 되었는지 확인하는 방법

작업 후 아래 주소를 **인터넷 브라우저 주소창에 그대로 붙여넣어** 보세요:

```
https://slab.sparkerp.co.kr/api/1.1/obj/sparklabinvestment?limit=1
```

- 데이터(글자·숫자 묶음)가 **보이면 성공** 입니다 → 저희에게 "됐어요" 한마디만 주세요.
- `Type not found`(404) → 아직 **3단계 Deploy(배포)** 가 안 된 것입니다.
- `Invalid or expired token`(401) → 토큰 방식으로 막혀 있고 토큰 문제입니다.

성공하면 저희가 나머지 연동 작업을 이어서 진행하겠습니다. 감사합니다!
